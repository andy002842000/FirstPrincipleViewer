# FirstPrincipleViewer

[English](README.md) · [繁體中文](README.zh-TW.md) · **日本語** · [한국어](README.ko.md)

**画面上の任意の領域を選択 → ローカル OCR → Gemini による「第一原理」からの解説をリアルタイムにストリーミング。**

軽量なクロスプラットフォームのデスクトップツール(Windows / macOS / Linux)。**BYOK(Bring Your Own Key)** 方式:自分の Google Gemini API キーをアプリ内で入力し、お使いのマシンに暗号化して保存します。

> 🔒 **プライバシーとコストを最優先に設計:** スクリーンショットがマシンの外に出ることはありません。LLM に送られるのは OCR で抽出した「テキスト」のみで、しかも安価なテキストモデルを使用します —— 高価なビジョン API は使いません。

---

## このツールを作った理由

AI エージェントはますます速くなり、エンジニアリングの情報量は爆発しています —— 人間が「読む」だけでも追いつけないほどで、ましてや吸収するのは困難です。目の前のすべての情報を素早く把握することが、どんどん難しくなっています。

FirstPrincipleViewer はその近道です:立ち止まって考える必要はなく、理解したい領域をただ枠で囲むだけ。中のテキストを読み取り、**リアルタイム**で「**第一原理から説明する**要約」に変換します —— 「何か」を流し読みするのではなく、「なぜか」をつかむために。

## 仕組み

```
領域を選択  →  フレームを取得  →  ローカル OCR(Tesseract)  →  テキスト  →  Gemini(テキストモデル)  →  解説をストリーミング
```

「**継続モニター**」モードは一定間隔で領域を再取得し、認識されたテキストが実際に変化したときだけ LLM を再び呼び出します。

### 方法 A — ビルド済みをダウンロード
[Releases ページ](https://github.com/andy002842000/FirstPrincipleViewer/releases) から、お使いの OS 向けの最新インストーラーを入手してください。

```bash

cd FirstPrincipleViewer

```

## 自分のキーを使う(BYOK)

1. <https://aistudio.google.com/apikey> で**無料**の Gemini API キーを取得します。
2. アプリを起動 → **⚙(設定)** ボタンをクリック → キーを貼り付け → **テスト** → **保存**。
3. キーは OS のキーストア(Windows DPAPI / macOS Keychain / Linux secret service)で暗号化して保存され、二度と表示されず、Google の API 以外へ送信されることもありません。

> 開発時は代わりにローカルの `.env` に `GEMINI_API_KEY` を置くこともできます(`.env.example` 参照)。アプリ内に保存したキーが常に優先されます。

## 使い方

1. **Ctrl / ⌘ + Shift + E** を押します(または「領域を選択」をクリック)。
2. 解説してほしいテキストを、どのモニターでもドラッグして囲みます。
3. 認識されたテキストと、ストリーミングされる第一原理からの解説がパネルに表示されます。
4. 「継続モニター」をオンにすると、領域の内容が変わるたびに更新され続けます。

## 設定

| 設定項目 | 意味 | デフォルト |
| --- | --- | --- |
| API キー | あなたの Gemini キー(暗号化保存) | — |
| モデル | Gemini テキストモデル | `gemini-3.1-flash-lite` |
| OCR 言語 | Tesseract の言語、例 `eng`、`eng+chi_tra` | `eng+chi_tra` |
| 解説言語 | 解説を書く言語 | `Traditional Chinese` |
| モニター間隔 | 再取得の間隔(ミリ秒) | `1500` |

## コスト(透明性)

お金がかかる唯一のステップは **Gemini のテキスト API 呼び出し**です。画面の取得、OCR(Tesseract.js)、変化検出はすべて**ローカルかつ無料**で動作し、**スクリーンショットがアップロードされることはありません**。送られるのは OCR したテキストだけです。

各「選択 → 解説」呼び出しは `入力トークン + 出力トークン` で課金されます:

- **入力** = システム指示(~120 トークン)+ プロンプトの包み(~30)+ 取得したテキスト
- **出力** = ストリーミングされる解説(通常はこちらが大きい)

デフォルトモデル **`gemini-3.1-flash-lite`** —— 価格は 2026-05 時点の有料ティア(必ず <https://ai.google.dev/pricing> で確認してください):

- 入力:**$0.25 / 100万トークン**
- 出力:**$1.50 / 100万トークン**
- **無料ティア**あり(レート制限付き) —— 個人の軽い利用なら実質 $0 のことが多いです。

### 1 回あたりの概算コスト

| 領域 | 入力トークン | 出力トークン | 1 回あたり |
| --- | --- | --- | --- |
| 小(1 文 / ラベル) | ~230 | ~400 | ~$0.0007 |
| 中(段落 / コード) | ~600 | ~800 | ~$0.0014 |
| 大(セクション全体) | ~1,650 | ~1,200 | ~$0.0022 |

おおよそ**1 回 $0.0007–$0.002** —— **US$1 で約 500–1,500 回**です。(トークン数は概算で、CJK のテキストはトークン化が異なります。)

「**継続モニター**」モードは OCR テキストが実際に**変化**したときだけ API を呼び出します —— 静的な領域を監視している間はローカル OCR を続けるだけで、新たな課金呼び出しは発生しません。

### コストを下げるには

- 設定でより安価なモデルを選ぶ。例:`gemini-2.5-flash-lite`(100万あたり 入力 $0.10 / 出力 $0.40)。
- 解説を短く保つ —— 出力トークンが費用の大半を占めます。
- 軽い利用なら無料ティアの範囲内に収める。

## プラットフォーム別の注意

- **macOS:** **画面収録**の許可が必要です(システム設定 → プライバシーとセキュリティ → 画面収録)。アプリが案内します。許可後は再起動してください。
- **Linux:** 利用可能な keyring / secret service がない場合、キーは平文保存にフォールバックします —— その際は設定ダイアログが警告します。画面キャプチャは X11 が最も安定。Wayland では PipeWire が必要です。
- **マルチモニター:** 対応 —— 各ディスプレイにそれぞれ選択オーバーレイが表示されます。1 回の選択は同一ディスプレイ内に収める必要があります。
